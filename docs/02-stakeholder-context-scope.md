# 2. System Context

ระบบ **Classroom & Lab Maintenance Reporting System (CLMRS)** เป็นระบบสำหรับรับแจ้งปัญหาภายในห้องเรียนและห้องปฏิบัติการของมหาวิทยาลัย ผู้ใช้งานสามารถแจ้งเหตุ พร้อมแนบรูปภาพและรายละเอียดของปัญหา ระบบจะส่งข้อมูลไปยังเจ้าหน้าที่ซ่อมบำรุงเพื่อตรวจสอบ ดำเนินการ และอัปเดตสถานะจนงานเสร็จสิ้น

ข้อมูลเข้า (Input)

- รายละเอียดปัญหา
- หมายเลขห้อง
- อาคาร
- ประเภทปัญหา
- รูปภาพประกอบ
- ข้อมูลผู้แจ้ง

ข้อมูลออก (Output)

- รายการแจ้งซ่อม
- สถานะการดำเนินงาน
- ประวัติการซ่อม
- รายงานสรุป

# 3. Scope Statement

## In Scope

| Area | Included capability | Rationale |
|---|---|---|
| User Management | Login และจัดการบัญชีผู้ใช้ | เพื่อระบุตัวตนผู้ใช้งาน |
| Maintenance Request | แจ้งปัญหาพร้อมรูปภาพ | เป็นความสามารถหลักของระบบ |
| Work Management | เจ้าหน้าที่รับงานและอัปเดตสถานะ | ใช้ติดตามการดำเนินงาน |
| Notification | แจ้งเตือนเมื่อสถานะเปลี่ยน | ผู้ใช้ทราบความคืบหน้า |
| Dashboard | แสดงสถิติและรายงาน | สนับสนุนการตัดสินใจของผู้บริหาร |

## Out of Scope

| Area | Excluded capability | Reason |
|---|---|---|
| Online Payment | ระบบชำระเงิน | ไม่เกี่ยวข้องกับการแจ้งซ่อม |
| Inventory Management | จัดการคลังอะไหล่ | อยู่นอกขอบเขตโครงการ |
| Procurement | ระบบจัดซื้อวัสดุ | เป็นระบบของหน่วยงานอื่น |
| Mobile Application | แอปพลิเคชันมือถือ | พัฒนาเฉพาะ Web Application |

# 5. Ethics, Privacy and Accessibility Considerations

- **ข้อมูลส่วนบุคคลที่ระบบอาจเกี่ยวข้อง**
  - ชื่อผู้ใช้งาน
  - รหัสนักศึกษา
  - อีเมลมหาวิทยาลัย
  - รูปภาพที่แนบ

- **ความเสี่ยงด้านสิทธิ์การเข้าถึง/ความปลอดภัย**
  - ผู้ใช้ที่ไม่ได้รับอนุญาตอาจเข้าถึงข้อมูลได้
  - ข้อมูลการแจ้งซ่อมอาจถูกแก้ไขโดยไม่ได้รับอนุญาต
  - ต้องมีระบบกำหนดสิทธิ์ (Role-Based Access Control)

- **การเข้าถึงสำหรับผู้ใช้หลากหลายกลุ่ม**
  - รองรับการใช้งานผ่านคอมพิวเตอร์และโทรศัพท์มือถือ
  - ใช้ตัวอักษรที่อ่านง่าย
  - ปุ่มและเมนูมีขนาดเหมาะสม
  - รองรับผู้ใช้ที่มีข้อจำกัดด้านการมองเห็นด้วยข้อความที่ชัดเจน

- **ข้อควรระวังด้านจริยธรรม**
  - ไม่เปิดเผยข้อมูลส่วนบุคคลโดยไม่จำเป็น
  - ใช้ข้อมูลเพื่อการซ่อมบำรุงเท่านั้น
  - จัดเก็บข้อมูลอย่างปลอดภัยตามนโยบายของมหาวิทยาลัย

# 6. Baseline Scope Decision

ทีมกำหนด Baseline ของระบบในสัปดาห์ที่ 2 ให้ประกอบด้วย

- ระบบ Login
- ระบบแจ้งซ่อมพร้อมแนบรูปภาพ
- ระบบติดตามสถานะงาน
- ระบบจัดการงานของเจ้าหน้าที่
- Dashboard แสดงสถิติการแจ้งซ่อม
- ระบบแจ้งเตือนสถานะงาน

ส่วนฟังก์ชันการจัดการคลังอะไหล่ การจัดซื้อ และ Mobile Application จะไม่นำมาพัฒนาในโครงการนี้ และอาจพิจารณาเป็นส่วนขยายในอนาคต โดยอ้างอิงจาก Decision Log ของทีม
