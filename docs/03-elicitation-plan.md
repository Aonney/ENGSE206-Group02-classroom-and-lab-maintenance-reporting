# 1. วัตถุประสงค์การเก็บความต้องการ

# 3. Interview / Workshop Guide

## Opening Script

> สวัสดีครับ/ค่ะ พวกเราเป็นนักศึกษารายวิชา ENGSE206 กำลังศึกษาความต้องการสำหรับโครงการระบบแจ้งซ่อมและติดตามการบำรุงรักษาห้องเรียนและห้องปฏิบัติการ
>
> การสัมภาษณ์ครั้งนี้มีวัตถุประสงค์เพื่อทำความเข้าใจขั้นตอนการทำงาน ปัญหา และความต้องการของผู้ใช้งาน ข้อมูลทั้งหมดจะใช้เพื่อการศึกษาเท่านั้น และจะไม่นำข้อมูลส่วนบุคคลหรือชื่อจริงไปเผยแพร่
>
> ขออนุญาตบันทึกคำตอบเพื่อนำไปสรุปความต้องการของระบบได้หรือไม่ครับ/คะ

- หากแก้ได้เพียงสิ่งเดียว คุณอยากให้แก้อะไรก่อน เพราะอะไร?
- มีข้อเสนอแนะเพิ่มเติมเกี่ยวกับระบบหรือขั้นตอนการแจ้งซ่อมหรือไม่?

# 4. Consent / Privacy Plan

- ไม่ใช้ชื่อจริงหรือข้อมูลที่สามารถระบุตัวบุคคลได้ในรายงาน
- ขออนุญาตก่อนบันทึกข้อมูลหรือถ่ายภาพ
- หากมีภาพประกอบ จะหลีกเลี่ยงใบหน้าและข้อมูลส่วนบุคคล
- ใช้ข้อมูลเพื่อการศึกษาในรายวิชาเท่านั้น
- ใช้ข้อมูลจำลองเมื่อจำเป็น

# 5. Elicitation Risks

| Risk | Mitigation |
|---|---|
| Stakeholder ไม่สะดวกให้สัมภาษณ์ | ใช้ Role-play หรือจำลองสถานการณ์ตามที่อาจารย์กำหนด |
| เวลาสัมภาษณ์จำกัด | เตรียมคำถามล่วงหน้าและเลือกเฉพาะคำถามสำคัญ |
| ได้ข้อมูลไม่ครบ | สัมภาษณ์มากกว่า 1 Stakeholder และตรวจสอบข้อมูลซ้ำ |
| ผู้ให้สัมภาษณ์เข้าใจคำถามไม่ตรงกัน | อธิบายคำถามเพิ่มเติมและยกตัวอย่างประกอบ |
| ข้อมูลเป็นความคิดเห็นส่วนบุคคล | เปรียบเทียบข้อมูลจากหลาย Stakeholders ก่อนสรุปผล |
