## Rehearsal Summary

### Scenario used

อาจารย์ผู้สอนพบว่าโปรเจคเตอร์ในห้องสอบเปิดไม่ติดก่อนเริ่มสอบ 15 นาที จึงต้องแจ้งซ่อมด่วน ในขณะเดียวกันมีนักศึกษาแจ้งเก้าอี้ห้องเลกเชอร์ชำรุด 1 ตัวผ่านช่องทางเดียวกันในวันเดียวกัน เจ้าหน้าที่อาคารสถานที่ต้องตัดสินใจว่าเคสใดควรจ่ายงานให้ช่างก่อน โดยที่ระบบปัจจุบันยังไม่มีกลไกจัดลำดับความสำคัญหรือแจ้งสถานะกลับให้ผู้แจ้งทราบ (เชื่อมโยงกับ **OQ-01** และ **OQ-04**)

### What was unclear / leading

ตรงตามผลจริงจาก Rehearsal (สอดคล้องกับตาราง "Question Rehearsal Revision" ใน `03-interview-guide.md` ส่วนที่ 4):

- **Q-03** — ผู้ตอบให้ข้อมูลไม่ครบว่าต้องใช้ข้อมูลอะไรบ้างในการรับงาน คำถามเดิมยังไม่ขอตัวอย่างข้อมูลจริง
- **Q-04** (OQ-01) — ผู้ตอบให้ความหมายของ "งานเร่งด่วน" แตกต่างกัน ทำให้ยังไม่ทราบเกณฑ์ที่ใช้จริง
- **Q-05** (OQ-02, OQ-06) — ผู้ตอบกล่าวถึงการส่งต่องาน แต่ไม่ได้อธิบายว่าใครเป็นผู้รับผิดชอบ ทำให้ Workflow ยังไม่ชัดเจน
- **Q-06** (OQ-04) — ผู้ตอบเริ่มเสนอช่องทางการแจ้งเตือนที่ต้องการเอง (เช่น LINE OA, Email) ก่อนอธิบายปัญหาที่แท้จริง เสี่ยงต่อ Solution Bias
- **Q-08** (OQ-05) — ผู้ตอบเสนอว่าต้องการ Dashboard แต่ยังไม่ระบุว่าต้องการข้อมูลหรือสถิติชนิดใด

### Follow-up questions added

1. (Q-03) "ช่วยยกตัวอย่างข้อมูลที่จำเป็นจากเหตุการณ์ล่าสุดได้หรือไม่?"
2. (Q-04 / OQ-01) "ช่วยยกตัวอย่างงานเร่งด่วนที่เคยเกิดขึ้นได้หรือไม่?"
3. (Q-05 / OQ-02, OQ-06) "ใครเป็นผู้ตัดสินใจส่งต่องาน และใครเป็นผู้ปิดงาน?"
4. (Q-06 / OQ-04) "ปัจจุบันมีปัญหาอะไรกับการติดตามสถานะ และเวลาใดที่ต้องการทราบข้อมูลมากที่สุด?"
5. (Q-08 / OQ-05) "ข้อมูลหรือสถิติใดที่ใช้ในการตัดสินใจบ่อยที่สุด และต้องการดูบ่อยเพียงใด?"

### Changes made to the interview guide

- Q-03: เพิ่มคำถามติดตามเพื่อขอตัวอย่างข้อมูลจริงจากเหตุการณ์ล่าสุด (Open Issue เพิ่มเติม นอกเหนือ OQ-01–06)
- Q-04: เพิ่มคำถามติดตามเพื่อขอตัวอย่างงานเร่งด่วนที่เกิดขึ้นจริง — ผูกกับ **OQ-01**
- Q-05: เพิ่มคำถามติดตามเพื่อระบุผู้ตัดสินใจส่งต่องานและผู้ปิดงาน — ผูกกับ **OQ-02, OQ-06**
- Q-06: ปรับคำถามให้ถามถึงปัญหาการติดตามสถานะก่อน แล้วจึงถามช่องทาง/เวลาที่ต้องการ เพื่อลด Solution Bias — ผูกกับ **OQ-04**
- Q-08: เพิ่มคำถามติดตามเพื่อระบุข้อมูล/สถิติที่ใช้ตัดสินใจบ่อยที่สุด — ผูกกับ **OQ-05**


### Evidence rules to use in Week 04

- ให้รหัสหลักฐาน เช่น `E-STAFF-01`, `E-STUDENT-02`, `E-LECTURER-01`, `E-TECH-01`
- บันทึก source และสถานการณ์/ห้องที่กล่าวถึงทุกครั้ง
- ระบุ type: Fact / Opinion / Preference / Assumption / Proposed Solution
- ระบุ Open Question ที่หลักฐานนั้นเกี่ยวข้อง (OQ-01 ถึง OQ-06 ตาม `03-elicitation-plan.md`) เพื่อ traceability
- ไม่แปลงคำพูดเป็น requirement ทันที; ให้นำไปวิเคราะห์เป็น candidate requirement พร้อม traceability กลับไปยัง Open Questions ใน `CASE_CARD.md`
